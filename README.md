# Convert-to-Webp

指定したディレクトリ内の画像ファイルを再帰的に検索し、WebP形式に一括変換するPythonツールです。
GUIによるフォルダ選択と進捗表示、マルチプロセス処理による高速な変換をサポートしています。

## 特徴

* **GUI操作**: `tkinter` を使用したフォルダ選択ダイアログとプログレスバーによる直感的な操作が可能です。
* **高速処理**: `ProcessPoolExecutor` を使用したマルチプロセス並列処理を行います。
* **CPU負荷制御**: 設定により、擬似的なCPU使用率の制限（スリープ挿入）や、CPUアフィニティの設定を行います。
* **ログ出力**: 処理結果やエラー情報をログファイルに記録します。

## 必要要件

* Python 3.9 以上
* 以下のPythonライブラリ:
    * `Pillow` (画像処理)
    * `psutil` (プロセス・システム管理)

## インストール

以下のコマンドで依存ライブラリをインストールしてください。

```bash
pip install Pillow psutil
```

## 使い方

1. `src/main.py` を実行します。
   ```bash
   python src/main.py
   ```
2. フォルダ選択ダイアログが表示されるので、画像ファイルが含まれるフォルダを選択してください。
3. ファイルの検索が行われ、変換処理が開始されます。進捗バーで状況を確認できます。
4. 変換後の `.webp` ファイルは、元の画像ファイルと同じディレクトリに保存されます（元のファイルは削除されません）。
5. 実行ログはカレントディレクトリの `Result.log` に出力されます。

## 設定 (src/settings.py)

`src/settings.py` を編集することで、以下の動作を変更できます。

* **LOG_LEVEL**: ログの出力レベル (デフォルト: "INFO")
* **FILE_TYPES**: 変換対象とする画像の拡張子リスト
* **QUALITY**: WebP変換時の品質 (0-100, デフォルト: 80)
* **MAX_WORKERS**: 同時実行するプロセス数 (デフォルト: 14)
* **CPU_LOAD**: 目標とするCPU負荷率。この値に基づき処理間にスリープを挟みます (デフォルト: 80)

## 注意事項

* ソースコード内で `ZoneInfo("Asia/Tokyo")` を使用しているため、タイムゾーンは日本時間でログ記録されます。
* Windows環境でのプロセス親和性（CPU Affinity）設定が含まれています。
